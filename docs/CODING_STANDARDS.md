# Coding Standards

Always follow SOLID.

Prefer composition over inheritance.

Never duplicate business logic.

Keep methods under approximately 30 lines when practical.

Controllers should remain thin.

Services should contain business rules.

Repositories should only retrieve data.

Use constructor injection.

Use DTOs.

Never expose Entity directly.

Avoid static utility classes unless justified.

Favor readability over clever optimizations.

Every new class should have a single responsibility.

Generate code that would pass a professional code review.
